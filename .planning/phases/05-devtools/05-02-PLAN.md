**Implementation:**
```csharp
public class DevToolsMiddleware<TState> : IMiddleware<TState> where TState : class
{
    private readonly IDevToolsStore _devToolsStore;

    public DevToolsMiddleware(IDevToolsStore devToolsStore)
    {
        _devToolsStore = devToolsStore;
    }

    public bool OnBeforeChange(MiddlewareContext<TState> context)
    {
        // DevTools never blocks state changes
        return true;
    }

    public void OnAfterChange(MiddlewareContext<TState> context)
    {
        // Don't record during time-travel
        if (_devToolsStore.IsTimeTraveling)
            return;

        _devToolsStore.RecordStateChange(
            context.StoreType,
            context.OldState,
            context.NewState,
            context.ActionName,
            context.Timestamp);
    }
}
```

Add XML documentation explaining:
- Never blocks changes (OnBeforeChange always returns true)
- Records to DevToolsStore for history tracking
- Skips recording when time-traveling to prevent duplicate entries

The middleware needs a reference to Bustand core project for IMiddleware<TState> and MiddlewareContext<TState>. Verify the project reference exists.

Update AddBustandDevTools to register services with environment protection.

**Implementation:**
```csharp
public static IServiceCollection AddBustandDevTools(
    this IServiceCollection services,
    IHostEnvironment environment)
{
    if (!environment.IsDevelopment())
    {
        Console.WriteLine("[Bustand DevTools] WARNING: DevTools should only be enabled in Development. " +
            "Skipping registration for security.");
        return services;
    }

    // Register DevToolsStore as Scoped (one per circuit/user)
    services.AddScoped<IDevToolsStore, DevToolsStore>();

    return services;
}
```

**Also add overload for convenience without environment (throws in production):**

{
    // For use when environment is not available - always registers

    return services;
}
```

/// <summary>
/// Gets whether DevTools middleware should be enabled.

internal bool DevToolsEnabled { get; private set; }

/// <summary>
/// Enables DevTools middleware for state change capture.
/// Call this only when DevTools services are registered (development environment).
/// </summary>
/// <returns>The options instance for chaining.</returns>
/// <remarks>
/// DevTools middleware captures all state changes for inspection in the DevTools page.
/// This should only be enabled in development environments.
/// </remarks>
/// <example>
/// <code>
/// builder.Services.AddBustand(options => options
///     .UseLogging()
///     .UseDevTools());
/// </code>
/// </example>
public BustandOptions UseDevTools()
{
    DevToolsEnabled = true;
    return this;
}
```

**Note:** The actual middleware registration happens in ServiceCollectionExtensions when it sees DevToolsEnabled=true. The middleware type `DevToolsMiddleware<TState>` is in Bustand.DevTools, so we need to handle this via reflection or by having the DevTools package extend the registration.

For now, UseDevTools() just sets the flag. The actual middleware wiring will need to be done by DevTools package's AddBustandDevTools, which should check if UseDevTools was called and add the middleware type to the pipeline.
