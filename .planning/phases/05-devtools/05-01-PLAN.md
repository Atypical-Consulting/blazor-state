  <action>
Implement DevToolsStore class that manages state history per store.

**Implementation details:**
- Private `Dictionary<string, List<StateSnapshot>> _history` for per-store history
- Private `Dictionary<string, int> _currentIndex` for time-travel position tracking
- Private `Dictionary<string, IStore> _stores` for store instance references (needed for time-travel)
- Private `JsonSerializerOptions _jsonOptions` with WriteIndented=true, ReferenceHandler=IgnoreCycles, PropertyNamingPolicy=CamelCase
- Private `const int MaxHistoryLength = 100` per CONTEXT.md decision
- Private `bool _isTimeTraveling` flag

**RecordStateChange method:**
1. If `_isTimeTraveling` is true, return immediately (don't record during time-travel)
2. Get storeName from `storeType.Name`
3. Initialize history list if store not seen before
4. Truncate future history if currentIndex < history.Count - 1 (branching after time-travel)
5. Enforce MaxHistoryLength limit (remove oldest)
6. Serialize newState to JSON with try/catch (fallback to error message on failure)
7. Create StateSnapshot and add to history
8. Update currentIndex to latest
9. Fire StateHistoryChanged event

**GetHistory/GetCurrentSnapshot:**
- Return empty/null if store not found
- GetCurrentSnapshot uses currentIndex to support time-travel position

**JumpToState method:**
- Placeholder for now (will be wired in later plan)
- Set _isTimeTraveling = true, then false after operation
- Update _currentIndex for the store

**RegisterStore method (internal):**
- Allows middleware setup to register store instances
- Stores in _stores dictionary

Use `System.Text.Json` for serialization per RESEARCH.md recommendations.
