<task type="auto">
  <name>Task 2: Create Directory.Build.targets with DefaultDocumentation configuration</name>
  <files>Directory.Build.targets</files>
  <action>
Create Directory.Build.targets to configure DefaultDocumentation:

**Directory.Build.targets**:
```xml
<Project>
  <!--
    DefaultDocumentation Configuration
    Generates markdown API docs from XML documentation comments.
    Output: docs/api/

    Documentation is generated during build for library projects.
    See: https://github.com/Doraku/DefaultDocumentation
  -->

  <ItemGroup Condition="'$(IsPackable)' != 'false'">
    <PackageReference Include="DefaultDocumentation" Version="1.2.2" PrivateAssets="all" />
  </ItemGroup>

  <PropertyGroup Condition="'$(IsPackable)' != 'false'">
    <!-- Output directory (relative to solution root) -->
    <DefaultDocumentationFolder>$(MSBuildThisFileDirectory)docs/api</DefaultDocumentationFolder>

    <!-- Only document public API -->
    <DefaultDocumentationGeneratedAccessModifiers>Public</DefaultDocumentationGeneratedAccessModifiers>

    <!-- Skip items without documentation -->
    <DefaultDocumentationIncludeUndocumentedItems>false</DefaultDocumentationIncludeUndocumentedItems>

    <!-- Generate pages for namespaces, types, and members -->
    <DefaultDocumentationGeneratedPages>Namespaces,Types,Members</DefaultDocumentationGeneratedPages>

    <!-- File naming: use full type name for clarity -->
    <DefaultDocumentationFileNameMode>FullName</DefaultDocumentationFileNameMode>
  </PropertyGroup>
</Project>
```

Key decisions:
- Condition `'$(IsPackable)' != 'false'` ensures only library projects generate docs (not test projects)
- PrivateAssets="all" prevents DefaultDocumentation from becoming a runtime dependency
- Only public API documented (internal implementation hidden)
- Undocumented items skipped (encourages adding XML docs)
- Full type names in file names for clarity

Per RESEARCH.md: DefaultDocumentation 1.2.2 is the standard for API reference generation.
  </action>
  <verify>
File Directory.Build.targets exists in repository root.
Contains DefaultDocumentation package reference.
Contains DefaultDocumentationFolder property.
  </verify>
  <done>Directory.Build.targets created with DefaultDocumentation configuration.</done>
